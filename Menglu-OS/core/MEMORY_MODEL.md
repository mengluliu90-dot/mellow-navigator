# Memory Model

## Purpose

Define what the system should remember and how it should use context without storing sensitive evidence in this public repository.

## Memory buckets

### 1. Communication preferences

- written communication preferred
- avoid phone contact where possible
- allow extra processing time
- use clear and structured wording
- one issue at a time where possible

### 2. Accessibility needs

- reasonable adjustments
- support person involvement
- simple written summaries
- reduced time pressure
- predictable steps

### 3. Case status

Use non-sensitive summaries only:

- case title
- organisation
- status
- next step
- deadline
- risk level

### 4. Evidence map

Store categories and references only.

Do not store private evidence itself in this public repository.

### 5. Templates

Reusable wording for emails, requests, complaints, and follow-ups.

## Memory use rules

- Use memory to reduce repetition.
- Check whether new information links to existing issues.
- Avoid asking Menglu for information already available in the current working context.
- Do not overstate memory as evidence.
- If uncertain, label as Needs clarification.

## Public repository boundary

This repository should contain operating instructions and non-sensitive templates only.

Private records must remain outside this repository.
