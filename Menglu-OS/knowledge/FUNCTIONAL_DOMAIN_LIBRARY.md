# Functional Domain Library

## Purpose

This file defines the reusable functional domains used by Menglu OS when organising evidence, drafting advocacy materials, or checking whether a new statement duplicates existing information.

This is an index structure only. Do not store private evidence or detailed personal records in this public repository.

## Core rule

Functional evidence should describe what happens in real life, what support is required, what goes wrong without support, and what the consequences are.

Use these categories as an index only. Do not store live booking details, private travel records, personal identifiers, or detailed medical evidence in this public repository.

## Evidence quality checklist

For each domain, prefer examples that show:

1. The task or situation.
2. The barrier.
3. Support required.
4. What happens without support.
5. Delayed consequences.
6. Whether the issue happens reliably, repeatedly, safely, to an acceptable standard, and within a reasonable time.

## Real-world evidence rule

When available, use documented real-world incidents to support functional descriptions.

Examples of evidence types include:

- professional reports
- appointment outcomes
- written correspondence
- service-access barriers
- financial or administrative incidents
- assistive equipment records
- support sessions
- diaries or structured statements

Do not store the private details here. Store only the domain category and non-sensitive index reference.
